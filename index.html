Responsive Redesign

In order to investigate responsive design, I chose a website I thought had a flawed interface and redesigned it to be fix the flaws I saw and design a responsive site.

Identifying Usability Problems

I first chose a website and analyzed its interface and what I thought could be improved about it. I paid special attention to usability, learnability, memorability, and its agreement with users' conceptual models.

1. Picking a Website

I picked the Linux Manual Page for the cat command. I chose this website because I used the manual pages a lot in my CS courses and always thought that it was confusing and poorly formatted; It was a necessary resource that I really disliked using and thought would be an interesting subject to analyze.

2. Finding Problems

The main issues with the page is it's lack of responsiveness and unintuitive organization:

  • unresponsive for mobile layouts or varying widths
    • excessive whitespace on desktop
    • inconsistent wrapping (overflow for the content, but wrapping on header, footer, and navigation menu)
  • lack of intuitive text size hierarchy, especially without color
  • “top” link is unclear that it is an actual link without the color

3. Accessibility

In terms of accessibility, the biggest problems had to do with page layout, which I agreed with. There were no page regions within the html (e.g. header, footer) and layout tables were used that didn't actually have content meant for tables, which is confusing for screen readers. One issue I didn't agree with was the redundant links, which sometimes I felt were useful for clarity.
